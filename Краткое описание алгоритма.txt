# Краткое описание алгоритма поиска аномальных респондентов

## Общая идея

Цель алгоритма — выявление респондентов, которые создают аномально высокий вклад в показатели OTS отдельных брендов в конкретные дни наблюдения. Поиск аномалий выполняется на уровне комбинации «респондент – день – бренд», а итоговое удаление производится на уровне пары «респондент – день», как требуется в условии задания.

Алгоритм ориентирован на обнаружение экстремально высокой активности, способной искусственно формировать пики OTS брендов. Низкая активность не рассматривается как аномалия.

## Расчет дневного OTS

Для каждой комбинации респондента, бренда и дня рассчитывается показатель daily_ots как произведение веса респондента на количество запросов данного респондента по данному бренду за выбранный день.

Таким образом учитываются как интенсивность активности пользователя, так и его вклад в репрезентативность панели.

## Используемые признаки

Для оценки аномальности используются несколько независимых характеристик.

### Аномальность относительно бренда

Для каждого бренда рассчитывается robust z-score по распределению значений daily_ots. В качестве центральной тенденции используется медиана, а в качестве меры разброса — медианное абсолютное отклонение (MAD), что делает оценку устойчивой к выбросам.

Для брендов с недостаточным количеством наблюдений аномальность оценивается относительно распределения внутри соответствующей категории товаров. Это позволяет получать более стабильные оценки для редких брендов.

Данный показатель отражает, насколько активность конкретного респондента отличается от типичного поведения пользователей данного бренда.

### Доля респондента в OTS бренда за день

Для каждой записи рассчитывается доля респондента в общем OTS бренда за конкретный день.

Высокое значение этого показателя означает, что респондент оказывает существенное влияние на итоговый показатель бренда и потенциально формирует искусственный пик активности.

### Отклонение от собственной истории активности

Для каждого пользователя рассчитывается отношение его суммарного OTS за текущий день к медианному значению его дневной активности за весь период наблюдения.

Этот показатель позволяет выявлять случаи, когда пользователь ведет себя нетипично по сравнению со своим обычным уровнем активности.

### Абсолютный уровень активности

Дополнительно учитывается само значение daily_ots. Это позволяет отдавать приоритет случаям, которые не только статистически необычны, но и обладают значительным влиянием на итоговые показатели.

## Расчет итогового score

Для каждой записи формируется интегральный показатель score, который объединяет:

* степень отклонения относительно бренда;
* отклонение от собственной истории активности;
* долю респондента в бренд-дневном OTS;
* абсолютный уровень активности.

Компоненты объединяются мультипликативно. Благодаря этому высокий score возникает только тогда, когда одновременно наблюдаются несколько признаков аномального поведения. Если хотя бы один из ключевых признаков выражен слабо, итоговая оценка существенно снижается.

Таким образом алгоритм концентрируется на действительно подозрительных случаях, а не на отдельных случайных отклонениях.

## Выбор порогов

Порог аномальности рассчитывается автоматически отдельно для каждой товарной категории.

Для распределения значений score внутри категории вычисляются квартильные характеристики, после чего используется правило межквартильного размаха (IQR):

порог = Q3 + 3 × IQR.

Такой подход позволяет учитывать различия между категориями и автоматически адаптироваться к структуре данных без ручной настройки фиксированного количества удаляемых наблюдений.

Для небольших категорий используется глобальный порог, рассчитанный по всей выборке.

## Дополнительные условия отбора

Высокого значения score недостаточно для признания записи аномальной.

Дополнительно проверяются следующие условия:

* высокая степень отклонения относительно бренда;
* существенное превышение типичной активности самого пользователя;
* высокий уровень daily_ots относительно своей категории;
* превышение рассчитанного порога аномальности.

Использование нескольких независимых критериев позволяет снизить вероятность ложных срабатываний.

## Формирование итогового списка аномалий

Если запись признается аномальной хотя бы по одному бренду в конкретный день, в итоговый список включается соответствующая пара «SubjectID – researchdate».

При последующей очистке удаляются все активности данного респондента за выбранный день независимо от бренда, что соответствует требованиям задания.

## Обоснование выбранного подхода

Предложенный алгоритм объединяет информацию о поведении конкретного бренда, истории активности самого пользователя и его влиянии на итоговые показатели бренда.

Использование устойчивых статистик на основе медианы и MAD делает алгоритм менее чувствительным к уже существующим выбросам. Применение адаптивных порогов на основе межквартильного размаха позволяет автоматически подстраиваться под различные категории товаров и использовать решение на новых данных без дополнительной настройки.

В результате алгоритм направлен на выявление респондентов, которые создают экстремальные бренд-дневные пики при минимальном количестве удалений и минимальной потере общего объема OTS.
